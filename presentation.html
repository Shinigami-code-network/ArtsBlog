Arts Playground — Project Summary

A lightweight storytelling site with admin tools and image uploads

Key Features

  • Public feed of stories with image support and full-story pages
  • Admin area: login, dashboard, create & delete stories
  • Image upload validation, size limit, safe filenames, uploads lock-down
  • Improved visuals: Google Fonts, responsive cards, admin theme
  • Error logging (no raw DB errors shown to users)

Presentation Checklist

  • Start XAMPP: sudo /opt/lampp/lampp start
  • Open: http://localhost/artsplayground
  • Admin panel: /admin/login.php
  • Use the demo admin credentials (or create a user in DB)
Tip: Use full-screen browser and the large card sections for clean screenshots.

Security & Hardening

  • Uploaded files validated with finfo + getimagesize()
  • Upload size limited to 3MB and PHP execution blocked in uploads/.htaccess
  • Database exceptions are logged to logs/app.log (not echoed)
  • Prepared statements used for DB inserts/queries
Consider adding CSRF tokens and stricter input validation before production.

Quick DB Schema (for demo)

CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(100) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  is_admin TINYINT(1) DEFAULT 0
);

CREATE TABLE stories (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  title VARCHAR(255) NOT NULL,
  content TEXT NOT NULL,
  image_path VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
